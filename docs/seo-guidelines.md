# SEO Guidelines - L'Admissible

## Objectif

Construire un media SEO-first sans sacrifier la voix editoriale. Le site doit repondre clairement aux intentions de recherche autour des ecoles de commerce, des admissions, des classements, de Parcoursup, des concours, des programmes post-bac et du Programme Grande Ecole.

## Principes

- Une page = une intention principale.
- Un titre clair vaut mieux qu'un jeu de mots opaque.
- Le SEO doit structurer l'information, pas la rendre robotique.
- Les contenus evergreen doivent etre maintenables.
- Les pages doivent charger vite, surtout sur mobile.

## Structure URL

Recommandations:

- `/ecoles/nom-ecole/`
- `/admissions/admissions-paralleles/`
- `/parcoursup/ecoles-commerce-post-bac/`
- `/classements/classement-ecoles-commerce/`
- `/concours/bce/`
- `/programme-grande-ecole/guide/`
- `/vie-etudiante/logement-ecole-commerce/`

Regles:

- URLs courtes.
- Minuscules.
- Mots separes par des tirets.
- Pas de date dans l'URL sauf besoin editorial fort.
- Eviter les slugs trop vagues: `/article-123/`, `/news/important/`.

## Balises essentielles

Chaque page doit prevoir:

- Un `title` unique.
- Une meta description specifique.
- Un seul H1.
- Une structure H2/H3 logique.
- Une balise canonical.
- Des donnees Open Graph.
- Des donnees structurees quand le type de page s'y prete.

## Donnees structurees a prevoir

- `Article` ou `NewsArticle` pour les articles.
- `BreadcrumbList` pour le fil d'Ariane.
- `FAQPage` uniquement si la FAQ est visible et utile.
- `Person` pour les auteurs si une vraie page auteur existe.
- `Organization` pour le media.

## Templates SEO prioritaires

### Article d'actualite

- H1 specifique.
- Date de publication et date de mise a jour si besoin.
- Auteur.
- Chapo.
- Corps de texte structure.
- Liens internes vers guides ou fiches ecoles.
- Sources citees.

### Guide evergreen

- H1 aligne avec la requete principale.
- Chapo clair.
- Sommaire.
- Sections H2 orientees questions ou besoins.
- Encadres pratiques.
- FAQ utile.
- Date de mise a jour visible.

### Fiche ecole

- H1: nom de l'ecole.
- Resume rapide.
- Admissions.
- Programmes.
- Frais.
- Campus.
- Classements si disponibles et contextualises.
- Liens vers articles lies.

### Page classement

- Source et date du classement.
- Methode resumee.
- Tableau lisible sur mobile.
- Analyse des mouvements importants.
- Limites du classement.
- Liens vers fiches ecoles.

## Maillage interne

Chaque contenu doit proposer des liens vers:

- Une categorie mere.
- Un guide evergreen quand le sujet s'y rattache.
- Une fiche ecole si une ecole est citee.
- Un article d'analyse si l'actualite demande du contexte.
- Une page classement si la comparaison est pertinente.

Le maillage doit aider le lecteur, pas juste faire circuler des robots.

## Mots-cles

Types de requetes prioritaires:

- "ecole de commerce post bac"
- "classement ecoles de commerce"
- "admission parallele ecole de commerce"
- "concours ecole de commerce"
- "Programme Grande Ecole"
- "Parcoursup ecole de commerce"
- "meilleure ecole de commerce"
- "frais ecole de commerce"
- "oral ecole de commerce"
- "BCE"
- "Ecricome"

Les mots-cles doivent etre integres naturellement. Pas de bourrage. Google sait lire, et les lecteurs aussi.

## Meta title

Format possible:

- `Sujet principal: angle ou precision | L'Admissible`
- `Nom ecole: admissions, programmes, frais | L'Admissible`
- `Classement des ecoles de commerce 2026: analyse | L'Admissible`

Longueur cible:

- Environ 50 a 60 caracteres quand possible.

## Meta description

Objectif:

- Resumer la valeur de la page.
- Donner un angle.
- Integrer la requete principale naturellement.

Longueur cible:

- Environ 140 a 160 caracteres quand possible.

## Performance SEO

- Generer les pages statiquement autant que possible.
- Limiter le JavaScript client.
- Optimiser les images.
- Prevenir les sauts de mise en page.
- Utiliser des polices avec strategie de chargement sobre.
- Eviter les scripts tiers inutiles.
- Surveiller Core Web Vitals.

## Contenu duplique

- Eviter les fiches ecoles construites sur le meme texte avec le nom remplace.
- Differencier les pages proches par l'intention.
- Utiliser des canonicals quand necessaire.
- Ne pas creer de tags faibles avec une seule page sans valeur.

## Checklist avant publication

- Intention de recherche claire.
- H1 unique.
- Title et meta description uniques.
- Slug propre.
- Chapo utile.
- Intertitres lisibles.
- Liens internes pertinents.
- Sources ajoutees si faits precis.
- Images avec dimensions et texte alternatif.
- Donnees structurees quand utile.
- Aucune phrase de remplissage ou formulation trop IA.
